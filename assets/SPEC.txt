## 2. Deployment Configuration (Vercel + GitHub + Domain)
### 2.1 Vercel & Domain Binding Steps
1. Push all Next.js code to GitHub `main` branch, link repo to Vercel.
2. Enter Vercel Project → `Settings` → `Domains` → Add domain: `games.nanobananas.me`.
3. Copy Vercel provided CNAME record value.
4. Go to domain registrar DNS management, add new record:
   - Type: `CNAME`
   - Host: `games`
   - Value: Vercel CNAME address
5. Wait 5~30 mins for DNS validation to pass.

### 2.2 ads.txt Configuration (AdSense Required)
1. Deploy `ads.txt` on **main domain root** (`nanobananas.me`), subdomain will inherit automatically.
2. File content (replace with your AdSense publisher ID):
   ```txt
   google.com, pub-XXXXXXXXXXXXXXXX, DIRECT, f08c47fec0942fa0
   ```
3. Verify access: `https://nanobananas.me/ads.txt`

### 2.3 Running Limits
- No restrictions on front-end logic, local storage is allowed.
- **No back-end data crawling / game reverse engineering / memory reading**. All data is static & manually sorted.

## 5. UI & Visual Style Specification
### 5.1 Overall Tone (Match game dark mechanical style)
- Main Background: `#0F2620` (Dark Green) / `#1A1A1A` (Pure Black)
- Highlight Color: `#E6C200` (Yellow, for emphasis, button border & key data)
